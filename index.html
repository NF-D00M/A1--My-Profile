Project Idea

Overview

Cram Lock is a third party educational tool that possesses the ability to select and lock unproductive applications which cannot be unlocked until a user has spent dedicated time on other productive applications. The functionality of Cram Lock follows.

  1. From their device users select productive applications they wish to dedicate their time.
  2. Users select unproductive applications they wish to lock.
  3. Users input the time frame of how long they must spend on productive applications for unproductive applications to unlock.
    • Locked time is configurable.
  4. Users input how often unproductive applications reset to a locked status.
    • Lock Reset time is configurable.

Overview

The development of Cram Lock addresses three core phenomenon impacting society: the acceleration of information retrieval, the decline of literacy and unproductive applications.

Lundstrom (2003) argues that Information Technology is evolving at an exponential rate; Moore’s law states that computer processing power doubles every two years due to microchip transistors decreasing in size and the production of computers halving in cost. This indicates that if Moore’s Law continues its upward projection, then the speed of information retrieval via technology streams will follow its course. Information retrieval allows users to discover content and express knowledge through numerous online learning platforms with the online education market estimated to be at $350 Billion (Koksal,2020). It’s safe to say that paperback encyclopedias and dictionaries are now redundant thanks to Google’s Search Engine. There is no denying that the revolution of technology has improved information sharing but this comes with its disadvantages.

Machine learning algorithms are so accurate that they can identify misspelled words on smartphones and can predict sentences before a user has finished typing (Stoop & Van Den Bosch, 2019). Regrettably, Bowerman (2015) suggests that predictive text has led to the decline of written literacy; users do not learn from autocorrect and often confuse the correct use of grammar. This poses the question, if written literacy has declined since the introduction and acceleration of information sharing, which other education sectors are at threat? Today’s digital software on computers and modern synthesizers can store an entire library of instruments, users can access any sound by interacting with a Musical Instrument Digital Interface (MIDI) (Beckstead, 2001). Designed within MIDI is a function called Automatic Error Correction (AER), that automatically corrects diatonic harmony if users play outside of the key signature (Marinov 2020).

For example, if a user declares the key signature as G Major and then play’s an F, then MIDI will automatically correct the F to an F#Sharp

AER makes creating music more productive, but again this comes with a sacrifice to literacy and deep learning. The entire foundation of learning musical harmony is built around scales and their key signatures, similar to predictive text favoured over spelling. Henschen (2018) claims there is a tragic decay of music theory, and it doesn’t help that classical music in general only appeals to an older audience. Along with spelling could music theory become a forgotten art?

Unproductive applications can be considered applications that users receive no value, this phenomenon is known as scrolling (Lupinacci, 2021). Lupinacci contintues to delve further into the subject and suggests that these apps can be predominantly linked to social media which can cause anxiety and fatigue. Wouldn’t it be beneficial if unproductive applications weren’t available until users had declared that they have had a productive day?

It is Cram Lock’s endeavour to make online learning platforms a smartphone priority. Considering there are 6.64 billion smartphone users in the world (Turner 2022), almost one smartphone per person, this may indicate that there is an opportunity in the palm of every hand globally. The project will take advantage of iOS and Android and build an application that locks selected apps which cannot be unlocked until users have completed their dedicated time on other nominated apps.

Description

Cram lock operates on the function that nominated unproductive applications cannot be accessed until a user has spent the adequate time on other productive apps. Following the process control Cram Lock can be defined by the following business requirements.

  1. Select Productive Apps
  2. Select Unproductive Apps
  3. Input Lock Time
  4. Input Lock Reset Time

Select Productive Apps. Productive applications are apps that the user generates value. The perfect example would be DuoLingo; a language-learning platform where users ascend a tress of skills to practise vocabulary, grammar and pronunciation. Users will have access to specified productive applications and non-specified applications throughout their smartphone.

Select Unproductive Apps. Users then must select unproductive applications that provide little value. If Facebook was selected, then Facebook cannot be unlocked until a user has spent the dedicated time on productive applications. Locked unproductive apps will be blacked out on the user’s home screen. Additionally, notifications and alerts will be disabled until the applications are unlocked, this will mimic the ‘do not disturb function.

Input Lock Time. Users can input the duration of time they must spend on productive applications for unproductive applications to unlock. Input can be defined to the minute, hour, specific time, or day. Smartphones will access their own data to analyse how long a user has spent on each application. Locking applications can be configurable per app i.e. Facebook can be set to 1 hour and Instagram can be set to 30 minutes, therefore users are only required to spend 30 minutes on Duolingo before Instagram is unlocked but have to spend a 1 hour to access Facebook.

Input Lock Reset Time. The last step is that users must input the duration for the entire process to start over. Once unproductive applications have unlocked users must declare the duration until they become locked again. This can be configured manually the same as Lock Time. Users can input an hourly reset or can wait to the next calendar day.

Interface. Once the locked applications are set the interface home screen will display all locked apps with a countdown timer that users must spend on productive apps. Productive apps will be listed underneath next to a settings cog. New users will be greeted with an onboarding sequence detailing the 4-step process and an overview of the interface.
